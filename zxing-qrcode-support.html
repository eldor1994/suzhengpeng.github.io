Class Version.ECBlocks

Encapsulates a set of error-correction blocks in one symbol version. Most versions will use blocks of differing sizes within one version, so, this encapsulates the parameters for each set of blocks. It also holds the number of error-correction codewords per block since it will be the same across all blocks within one version.
在一个符号版本中封装一组纠错块。
在大多数版本中，同一个版本中将会使用不同大小的块，因此，在这一封装中封装了每一组 的块的参数。这也保存了每个块的纠错码字的数量，因为这在同一个版本中所有的块的是相同的。

handlePossibleCenter

This is called when a horizontal scan finds a possible alignment pattern. It will cross check with a vertical scan, and if successful, will, ah, cross-cross-check with another horizontal scan. This is needed primarily to locate the real horizontal center of the pattern in cases of extreme skew. And then we cross-cross-cross check with another diagonal scan.
当水平扫描发现可能存在的校正图形时调用。这将会使用垂直扫描进行交叉检查，如果成功，进行 另一次水平交叉检查。这主要是为了图像在极端倾斜的情况下确定真正的水平中心。之后使用对角扫描 进行交叉检查

If that succeeds the finder pattern location is added to a list that tracks the number of times each location has been nearly-matched as a finder pattern. Each additional find is more evidence that the location is in fact a finder pattern center 如果成功，则将位置探测图形的位置添加到一个追踪每次定位接近匹配为一个探测图形的次数的列表 每次多一次的发现将增加该位置实际上是位置探测图形的中心的证据。

Parameters:
stateCount - reading state module counts from horizontal scan 读取水平扫描的状态模块计数
i - row where finder pattern may be found 位置探测图形可能存在行
j - end of possible finder pattern in row 位置探测图形可能存在最终的行
pureBarcode - true if in “pure barcode” mode 在纯条形码模式是为true

Returns:
true if a finder pattern candidate was found this time 如果候选位置探测图形在此次中发现，则为true
